Not clean. I read all Rust source under `control-plane/src` and `cli/src` and found 4 real production bugs.

- High — harden-stage inference is nondeterministic because jobs are loaded with `ORDER BY round ASC, attempt ASC` only, so same-round `audit` and `revise` jobs can tie and be returned in unspecified order; the driver then uses `rfind(... round == record.round)` to decide which sub-stage just ran, which can pick the wrong verdict/output and dispatch the wrong next step. Refs: `control-plane/src/state/postgres.rs:515`, `control-plane/src/loop_engine/driver.rs:398`, `control-plane/src/loop_engine/driver.rs:419`
- High — branch reuse can overwrite diverged remote branch history: the reuse check only blocks when `base_ref` is an ancestor of `origin/<branch>`, so a diverged stale remote branch can still be force-reset locally and then force-pushed with `--force-with-lease`. Refs: `control-plane/src/git/mod.rs:210`, `control-plane/src/git/mod.rs:467`
- Medium — `nemo auth` normalizes the engineer name only for the secret name, but still sends the original `engineer` string to the API; the control plane rejects anything outside lowercase alnum + `-`, so common names with `_`, spaces, `.`, or uppercase fail despite the CLI’s normalization. Refs: `cli/src/commands/auth.rs:25`, `cli/src/commands/auth.rs:79`, `control-plane/src/api/handlers.rs:78`, `control-plane/src/api/handlers.rs:514`
- Medium — missing provider credential files are treated as warnings, not errors, so default `nemo auth` can succeed and print `Credential references registered.` even when only one provider was actually registered; later loops that need the missing provider still fail/re-enter reauth. Refs: `cli/src/commands/auth.rs:64`, `cli/src/commands/auth.rs:95`

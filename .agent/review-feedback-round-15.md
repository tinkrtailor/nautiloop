1. High — `control-plane/src/git/mod.rs:174-236`, `control-plane/src/git/mod.rs:463-469`
`create_branch()` only guards remote-branch conflicts in the local-branch-already-exists path. If local `branch` is absent but `origin/<branch>` already exists, branch creation succeeds locally and `create_pr()` then does `git push --force-with-lease -u origin <branch>`, which can overwrite an existing remote branch/PR for the same deterministic branch name.

2. High — `control-plane/src/config/repo.rs:15-18`, `control-plane/src/config/mod.rs:64-67`, `control-plane/src/api/handlers.rs:43-49`
Repo config defines `[repo].default_branch`, but runtime default-ref resolution uses `self.cluster.default_branch` instead. That means `/start` reads specs from `origin/<cluster.default_branch>` rather than the repo’s declared default branch, so repos whose real base branch is not the cluster default can read the wrong ref and branch from the wrong base.

3. Medium — `control-plane/src/api/handlers.rs:78-97`, `control-plane/src/api/handlers.rs:537-551`
Engineer validation is inconsistent between `/start` and `/credentials`. `/start` enforces `1..=63` chars; `/credentials` does not. A user can register credentials for a name that later gets rejected when starting a loop.

4. Medium — `control-plane/src/k8s/job_builder.rs:127-136`, `control-plane/src/k8s/job_builder.rs:171-181`
Secret volume names are generated by sanitizing and truncating secret refs to 58 chars. Two distinct secret names can collapse to the same derived volume name, producing duplicate `volumes[].name` / mount references and causing Kubernetes job creation to fail.

5. Low — `cli/src/main.rs:175-180`
Loopback IPv6 detection matches `Some("[::1]")`, but `url::Url::host_str()` returns `::1` without brackets. `--insecure` will be incorrectly rejected for `https://[::1]:...` URLs.

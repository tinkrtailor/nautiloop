Found 4 real production bugs in the Rust source.

- High - New local-only agent branches get misclassified as “remote deleted” and the loop is cancelled. `create_branch()` only creates a local branch in `control-plane/src/git/mod.rs:174`, while divergence detection returns `RemoteGone` if `origin/<branch>` does not exist in `control-plane/src/git/mod.rs:248`. The running-state handler treats that as remote branch deletion and cancels the loop in `control-plane/src/loop_engine/driver.rs:244`. Trigger: start a loop and let reconciliation run before the agent has pushed the branch.
- High - Actual remote branch deletions are not reliably detected because fetch does not prune stale refs. `fetch()` uses plain `git fetch origin` in `control-plane/src/git/mod.rs:243`, but deletion detection depends on `rev-parse origin/<branch>` failing in `control-plane/src/git/mod.rs:262`. A deleted remote branch can still exist in local remote-tracking refs, so pause/cancel logic in `control-plane/src/loop_engine/driver.rs:181` and `control-plane/src/loop_engine/driver.rs:374` can miss the deletion.
- Medium - Finished jobs can be converted into failures after normal Kubernetes TTL cleanup. Jobs are created with `ttl_seconds_after_finished: Some(300)` in `control-plane/src/k8s/job_builder.rs:205`, but any later `JobStatus::NotFound` is treated as external deletion/failure in `control-plane/src/loop_engine/driver.rs:285`. Trigger: successful job + delayed reconciliation/control-plane downtime longer than 5 minutes.
- Medium - Ship/harden merge behavior is not frozen per loop and can change mid-flight with config changes. New loops store `max_rounds_for_auto_merge: None` and `merge_strategy: None` in `control-plane/src/api/handlers.rs:138`, even though those fields exist on `LoopRecord` in `control-plane/src/types/mod.rs:251`. Later decisions use live config from `self.config` in `control-plane/src/loop_engine/driver.rs:510`, `control-plane/src/loop_engine/driver.rs:717`, and `control-plane/src/loop_engine/driver.rs:762`, so a config change or restart can alter merge behavior for already-started loops.
